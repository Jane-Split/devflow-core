## 1. 概述

### 1.1 什么是 DevFlow

DevFlow 是一个 AI 驱动的开发工作流编排工具，旨在自动化软件开发的整个生命周期。它能够：

- 自动分析项目结构和代码规范
- 智能拆分任务并管理依赖关系
- 支持多智能体并行开发
- 提供长期记忆和模式学习
- 自动化测试和回归测试

### 1.2 核心特性

| 特性 | 说明 |
|:---|:---|
| 完整生命周期 | 研究 → 分析 → 设计 → 拆分 → 开发 → 测试 → 修复 → 回归 |
| 自动分析 | 无需手动文档，自动提取项目特征 |
| 多项目支持 | 前端、后端、全栈、微服务 |
| 长期记忆 | 文件 + 向量双层存储 |
| 学习引擎 | 记录成功模式和常见错误 |
| 多工具兼容 | Cursor、Trae、Windsurf、Cline、Roo、Copilot |

### 1.3 系统要求

- Node.js >= 18.0.0
- npm >= 9.0.0 或 yarn >= 1.22.0

## 2. 安装

### 2.1 全局安装

全局安装后，可以在任何项目中使用 `devflow` 命令。

```bash
# 使用 npm
npm install -g @devflow-ai/core

# 使用 yarn
yarn global add @devflow-ai/core
```

验证安装：

```bash
devflow --version
```

### 2.2 项目级安装

项目级安装将 DevFlow 添加为开发依赖，适合团队共享配置。

```bash
# 使用 npm
npm install -D @devflow-ai/core

# 使用 yarn
yarn add -D @devflow-ai/core
```

### 2.3 从源码安装

```bash
# 克隆仓库
git clone https://github.com/your-org/devflow-core.git

# 进入目录
cd devflow-core

# 安装依赖
npm install

# 链接到全局
npm link
```

## 3. 快速开始

### 3.1 初始化项目

在项目根目录运行：

```bash
devflow init
```

这将创建以下文件和目录：

```
my-project/
├── .ai-memory/           # 记忆存储目录
│   ├── profiles/         # 项目画像
│   ├── requirements/     # 需求文档
│   ├── designs/          # 设计文档
│   ├── tasks/           # 任务卡片
│   ├── executions/       # 执行记录
│   └── learning/         # 学习数据
├── devflow.config.js     # 配置文件
└── AGENTS.md            # AI 代理规则
```

### 3.2 分析项目

分析项目结构，提取技术栈和代码规范：

```bash
# 基本分析
devflow analyze

# 详细分析（包含增量更新）
devflow research --incremental

# 强制刷新
devflow research --refresh
```

### 3.3 运行工作流

```bash
# 运行完整工作流
devflow run --requirement ./requirements.md

# 从指定阶段开始
devflow run --phase test --requirement ./requirements.md

# 自动确认所有步骤
devflow run --auto-approve --requirement ./requirements.md
```

### 3.4 查看状态

```bash
# 查看项目状态
devflow status

# JSON 格式输出
devflow status --json

# 详细信息
devflow status --verbose
```

## 4. CLI 命令详解

### 4.1 初始化命令

```bash
devflow init [options]
```

**选项：**

| 选项 | 说明 | 默认值 |
|:---|:---|:---|
| `--force` | 强制覆盖现有配置 | false |
| `--template <name>` | 使用指定模板 | default |
| `--project-name <name>` | 项目名称 | 目录名 |

**示例：**

```bash
# 使用默认模板初始化
devflow init

# 强制重新初始化
devflow init --force

# 指定项目名称
devflow init --project-name my-awesome-app
```

### 4.2 研究命令

```bash
devflow research [options]
```

**选项：**

| 选项 | 说明 | 默认值 |
|:---|:---|:---|
| `--refresh` | 强制刷新现有研究 | false |
| `--incremental` | 增量更新 | false |
| `--scope <scope>` | 分析范围 (frontend/backend/database/all) | all |

**示例：**

```bash
# 首次研究
devflow research

# 只分析前端
devflow research --scope frontend

# 增量更新
devflow research --incremental

# 强制刷新
devflow research --refresh
```

### 4.3 分析命令

```bash
devflow analyze [options]
```

**选项：**

| 选项 | 说明 | 默认值 |
|:---|:---|:---|
| `--output <path>` | 输出文件路径 | - |
| `--format <format>` | 输出格式 (json/markdown) | markdown |

**示例：**

```bash
# 基本分析
devflow analyze

# 输出到文件
devflow analyze --output ./analysis-report.md
```

### 4.4 运行命令

```bash
devflow run [options]
```

**选项：**

| 选项 | 说明 | 默认值 |
|:---|:---|:---|
| `--requirement <path>` | 需求文档路径 | - |
| `--phase <phase>` | 起始阶段 | research |
| `--end-phase <phase>` | 结束阶段 | regression |
| `--auto-approve` | 自动确认 | false |
| `--verbose` | 详细输出 | false |

**阶段选项：**

- `research` - 项目研究
- `analyze` - 需求分析
- `design` - 详细设计
- `split` - 任务拆分
- `dev` - 开发执行
- `test` - 测试执行
- `fix` - Bug 修复
- `regression` - 回归测试

**示例：**

```bash
# 运行完整工作流
devflow run --requirement ./requirements.md

# 从设计阶段开始
devflow run --phase design --requirement ./requirements.md

# 只运行到测试阶段
devflow run --phase research --end-phase test --requirement ./requirements.md

# 详细输出
devflow run --verbose --requirement ./requirements.md
```

### 4.5 测试命令

```bash
devflow test [options]
```

**选项：**

| 选项 | 说明 | 默认值 |
|:---|:---|:---|
| `--e2e` | 运行 E2E 测试 | false |
| `--integration` | 运行集成测试 | false |
| `--coverage` | 生成覆盖率报告 | false |
| `--headed` | 可视化浏览器模式 | false |

**示例：**

```bash
# 运行所有测试
devflow test

# 只运行单元测试
devflow test --integration false --e2e false

# 运行 E2E 测试（可视化）
devflow test --e2e --headed

# 生成覆盖率报告
devflow test --coverage
```

### 4.6 记忆命令

```bash
devflow memory <action> [options]
```

**操作：**

| 操作 | 说明 |
|:---|:---|
| `search <query>` | 搜索记忆 |
| `read <id>` | 读取记忆条目 |
| `write <category>` | 写入记忆 |
| `stats` | 显示记忆统计 |
| `export` | 导出记忆 |
| `import <path>` | 导入记忆 |
| `clear` | 清除记忆 |

**示例：**

```bash
# 搜索记忆
devflow memory search "React hooks"

# 查看统计
devflow memory stats

# 导出记忆
devflow memory export --output ./memory-backup.json
```

### 4.7 状态命令

```bash
devflow status [options]
```

**选项：**

| 选项 | 说明 | 默认值 |
|:---|:---|:---|
| `--verbose` | 显示详细信息 | false |
| `--json` | JSON 格式输出 | false |
| `--phase <phase>` | 显示特定阶段状态 | - |

**示例：**

```bash
# 基本状态
devflow status

# 详细信息
devflow status --verbose

# JSON 格式
devflow status --json
```

### 4.8 版本命令

```bash

```

显示 DevFlow 版本信息和依赖版本。

## 5. 配置文件

### 5.1 配置文件位置

DevFlow 按以下优先级加载配置：

1. **项目配置**: `./devflow.config.js`
2. **全局配置**: `~/.devflow/config.js`
3. **默认配置**: 内置默认值

### 5.2 完整配置示例

```javascript
// devflow.config.js
export default {
  // 项目信息
  project: {
    name: 'my-project',
    type: 'auto', // auto | frontend | backend | fullstack | microservice
    root: process.cwd(),
  },

  // AI 工具配置
  aiTool: {
    // 自动检测或指定工具: cursor | trae | windsurf | cline | roo | copilot
    type: 'auto',
    
    // 上下文窗口大小
    contextLimit: 128000,
    
    // 启用子代理
    enableSubagent: true,
    
    // 最大并行任务数
    maxParallelTasks: 3,
  },

  // 记忆配置
  memory: {
    // 记忆存储目录
    directory: '.ai-memory',
    
    // 启用向量索引
    enableVectorIndex: true,
    
    // 嵌入模型: keyword | local | openai
    embeddingModel: 'auto',
    
    // 相似度阈值
    similarityThreshold: 0.7,
    
    // OpenAI API 密钥（使用 openai 嵌入时需要）
    openaiApiKey: process.env.OPENAI_API_KEY,
  },

  // 工作流配置
  workflow: {
    // 要执行的阶段
    phases: [
      'research',
      'analyze',
      'design',
      'split',
      'dev',
      'test',
      'fix',
      'regression',
    ],
    
    // 自动确认阶段转换
    autoApprove: false,
    
    // 失败时停止
    stopOnFailure: true,
    
    // 最大重试次数
    maxRetries: 3,
  },

  // 测试配置
  testing: {
    // 单元测试框架
    unitTestFramework: 'auto', // auto | jest | vitest | mocha
    
    // 集成测试框架
    integrationTestFramework: 'auto',
    
    // E2E 测试框架
    e2eTestFramework: 'playwright', // playwright | cypress
    
    // 覆盖率阈值
    coverageThreshold: 80,
    
    // E2E 浏览器选项
    e2eBrowser: 'chromium', // chromium | firefox | webkit
    e2eHeaded: false,
  },

  // 日志配置
  logging: {
    level: 'info', // debug | info | warn | error
    file: '.ai-memory/devflow.log',
    maxFileSize: '10m',
  },
};
```

### 5.3 环境变量

可以在 `.env` 文件或 shell 环境中设置：

```bash
# .env
DEVFLOW_PROJECT_NAME=my-project
DEVFLOW_AI_TOOL=cursor
DEVFLOW_OPENAI_API_KEY=sk-xxx
DEVFLOW_LOG_LEVEL=debug
```

## 6. 工作流程

### 6.1 八阶段工作流

DevFlow 实现了完整的八阶段工作流：

```
┌─────────────┐
│  Research   │ ← 项目调研，了解项目背景
└──────┬──────┘
       ↓
┌─────────────┐
│   Analyze    │ ← 需求分析，提取关键需求
└──────┬──────┘
       ↓
┌─────────────┐
│   Design    │ ← 详细设计，生成设计文档
└──────┬──────┘
       ↓
┌─────────────┐
│   Split     │ ← 任务拆分，生成任务卡片
└──────┬──────┘
       ↓
┌─────────────┐
│    Dev      │ ← 并行开发，执行任务
└──────┬──────┘
       ↓
┌─────────────┐
│    Test     │ ← 测试执行，单元+集成+E2E
└──────┬──────┘
       ↓
┌─────────────┐
│    Fix      │ ← Bug 修复
└──────┬──────┘
       ↓
┌─────────────┐
│ Regression  │ ← 回归测试
└─────────────┘
```

### 6.2 阶段详解

#### Research 阶段

- 分析项目结构和文件组织
- 检测技术栈和依赖
- 提取代码规范和模式
- 生成项目画像

#### Analyze 阶段

- 读取需求文档
- 提取关键需求点
- 识别功能模块
- 生成需求分析报告

#### Design 阶段

- 基于需求生成设计方案
- 定义模块接口
- 规划数据模型
- 生成设计文档

#### Split 阶段

- 将设计拆分为任务
- 分析任务依赖
- 评估任务复杂度
- 生成任务卡片

#### Dev 阶段

- 按依赖顺序并行执行任务
- 调用 AI 工具进行开发
- 记录执行日志
- 实时同步进度

#### Test 阶段

- 生成单元测试
- 生成集成测试
- 运行 E2E 测试（浏览器自动化）
- 收集测试覆盖率

#### Fix 阶段

- 分析测试失败原因
- 生成修复任务
- 自动执行修复
- 验证修复结果

#### Regression 阶段

- 重新运行所有测试
- 对比测试结果
- 生成回归报告
- 确认最终交付

### 6.3 自定义工作流

可以跳过某些阶段或添加自定义阶段：

```javascript
// 只执行研究和开发
workflow: {
  phases: ['research', 'dev'],
}

// 添加自定义阶段
workflow: {
  phases: ['research', 'analyze', 'custom-phase', 'dev', 'test'],
}
```

## 7. 记忆系统

### 7.1 记忆架构

DevFlow 使用混合记忆系统：

```
┌─────────────────────────────────┐
│         MemoryManager            │
│         (统一接口)               │
└───────────────┬─────────────────┘
                │
        ┌───────┴───────┐
        ↓               ↓
┌───────────────┐  ┌───────────────┐
│ FileMemory    │  │ VectorIndex  │
│ (文件存储)     │  │ (向量索引)    │
└───────────────┘  └───────────────┘
```

### 7.2 记忆分类

| 分类 | 说明 |
|:---|:---|
| `project-profile` | 项目画像 |
| `requirement` | 需求文档 |
| `design` | 设计文档 |
| `task` | 任务卡片 |
| `task-card` | 任务详情 |
| `execution` | 执行记录 |
| `execution-log` | 执行日志 |
| `learning` | 学习数据 |
| `pattern` | 成功模式 |
| `pitfall` | 踩过的坑 |

### 7.3 三级嵌入策略

DevFlow 支持三种嵌入方式，按优先级自动降级：

| 级别 | 实现 | 依赖 | 说明 |
|:---|:---|:---|:---|
| 1 | KeywordEmbeddingProvider | 无 | 零依赖，使用 TF-IDF |
| 2 | LocalEmbeddingProvider | @xenova/transformers | 本地 ONNX 模型 |
| 3 | OpenAIEmbeddingProvider | openai | OpenAI API |

**配置：**

```javascript
memory: {
  // 自动选择（推荐）
  embeddingModel: 'auto',
  
  // 强制使用某种方式
  embeddingModel: 'keyword', // 零依赖
  embeddingModel: 'openai',  // 需要 API Key
}
```

### 7.4 记忆管理命令

```bash
# 搜索相关记忆
devflow memory search "React 表单验证"

# 读取特定记忆
devflow memory read design-auth-module

# 写入新记忆
devflow memory write pattern --name "表单验证模式" --content "..."

# 查看统计
devflow memory stats

# 导出记忆
devflow memory export --output ./memory-backup.json

# 导入记忆
devflow memory import ./memory-backup.json

# 清除所有记忆
devflow memory clear --force
```

## 10. Skill 系统

### 10.1 什么是 Skill 系统

DevFlow Skill 系统允许你在 Trae、Cursor 等 AI 工具中通过 `/command` 命令快速唤醒 DevFlow 工作流。无需记忆复杂的 CLI 命令，只需输入简单的斜杠命令即可。

### 10.2 支持的命令

| 命令 | 说明 | 适用场景 |
|:---|:---|:---|
| `/devflow-init` | 初始化项目 | 新项目首次使用 DevFlow |
| `/devflow-analyze` | 分析项目 | 了解项目结构和规范 |
| `/devflow-run` | 执行工作流 | 开始开发新功能 |
| `/devflow-test` | 运行测试 | 验证代码质量 |

### 10.3 在 Trae 中使用

#### 配置步骤

1. **安装 DevFlow**
   ```bash
   npm install -D @devflow-ai/core
   ```

2. **复制 Skill 配置**
   
   将 `.trae/skills.json` 复制到 Trae 的 Skill 目录：
   ```bash
   # macOS
   cp node_modules/@devflow-ai/core/.trae/skills.json ~/.trae/skills/devflow.json
   
   # Windows
   copy node_modules\@devflow-ai\core\.trae\skills.json %USERPROFILE%\.trae\skills\devflow.json
   ```

3. **重启 Trae**

#### 使用示例

在 Trae 的 AI 对话中输入：

```
/devflow-init --name my-project --type frontend
```

Trae 将自动执行初始化流程。

### 10.4 在 Cursor 中使用

#### 配置步骤

1. **安装 DevFlow**
   ```bash
   npm install -D @devflow-ai/core
   ```

2. **复制 Skill 配置**
   
   将 `.cursor/skills.json` 复制到 Cursor 的 Skill 目录：
   ```bash
   # macOS
   cp node_modules/@devflow-ai/core/.cursor/skills.json ~/.cursor/skills/devflow.json
   
   # Windows
   copy node_modules\@devflow-ai\core\.cursor\skills.json %USERPROFILE%\.cursor\skills\devflow.json
   ```

3. **重启 Cursor**

#### 使用示例

在 Cursor 的 Composer 或 Chat 中输入：

```
/devflow-run --phase design --requirement ./feature.md
```

Cursor 将执行设计阶段的工作流。

### 10.5 命令详解

#### `/devflow-init`

初始化 DevFlow 项目配置。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| `name` | string | 否 | 项目名称 |
| `type` | enum | 否 | 项目类型：frontend/backend/fullstack/microservice |
| `force` | boolean | 否 | 强制重新初始化 |

**示例：**

```
/devflow-init
/devflow-init --name my-app --type fullstack
/devflow-init --force
```

#### `/devflow-analyze`

深度分析项目技术栈和代码规范。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| `deep` | boolean | 否 | 执行深度分析 |
| `output` | string | 否 | 分析报告输出路径 |

**示例：**

```
/devflow-analyze
/devflow-analyze --deep
/devflow-analyze --output ./analysis.md
```

#### `/devflow-run`

执行完整的 8 阶段开发工作流。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| `phase` | enum | 否 | 执行阶段：all/research/analyze/design/split/dev/test/fix/regression |
| `parallel` | boolean | 否 | 启用并行开发 |
| `requirement` | string | 否 | 需求描述或需求文档路径 |

**示例：**

```
/devflow-run --requirement ./feature.md
/devflow-run --phase dev --parallel
/devflow-run --phase test --requirement "添加用户登录功能"
```

#### `/devflow-test`

执行单元测试、集成测试或 E2E 测试。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| `type` | enum | 否 | 测试类型：unit/integration/e2e/all |
| `pattern` | string | 否 | 测试文件匹配模式 |
| `watch` | boolean | 否 | 监听模式 |

**示例：**

```
/devflow-test
/devflow-test --type e2e
/devflow-test --type unit --watch
```

### 10.6 在代码中使用 Skill API

你也可以在代码中直接调用 Skill 系统：

```javascript
import { runSkill, listSkills, getSkillHelp } from '@devflow-ai/core/skills';

// 列出所有可用的 Skills
const skills = await listSkills();
console.log(skills);

// 获取 Skill 帮助
const help = await getSkillHelp('/devflow-init');
console.log(help);

// 执行 Skill
const result = await runSkill('/devflow-init', {
  name: 'my-project',
  type: 'frontend'
}, '/path/to/project');

console.log(result.output);
```

## 11. 最佳实践

### 11.1 项目初始化

- 始终在项目根目录运行 `devflow init`
- 检查生成的 `AGENTS.md` 并根据需要修改
- 首次运行 `devflow research` 生成完整项目画像

### 11.2 需求文档

准备清晰的需求文档有助于 DevFlow 更好地工作：

```markdown
# 需求文档示例

## 功能需求

### FR-001: 用户登录

**描述**: 用户可以使用邮箱和密码登录系统

**验收标准**:
- [ ] 支持邮箱密码登录
- [ ] 显示错误提示
- [ ] 登录成功后跳转首页
- [ ] 记住登录状态

**优先级**: 高

### FR-002: 用户注册

**描述**: 新用户可以注册账号

**验收标准**:
- [ ] 邮箱格式验证
- [ ] 密码强度检查
- [ ] 注册成功后自动登录
```

### 11.3 迭代开发

```bash
# 1. 分析需求变化
devflow analyze --incremental

# 2. 重新设计
devflow run --phase design --end-phase split

# 3. 增量开发
devflow run --phase dev --end-phase test
```

### 11.4 团队协作

1. 将 `.ai-memory/` 添加到 `.gitignore`
2. 将 `devflow.config.js` 加入版本控制
3. 新成员运行 `devflow research` 生成本地记忆

## 12. 故障排除

### 12.1 常见问题

**Q: `devflow: command not found`**
```bash
# 重新链接
npm unlink -g @devflow-ai/core
npm install -g @devflow-ai/core
```

**Q: 记忆搜索返回空结果**
```bash
# 重建向量索引
devflow memory update-index

# 或强制刷新
devflow research --refresh
```

**Q: E2E 测试失败**
```bash
# 安装 Playwright 浏览器
npx playwright install

# 可视化调试
devflow test --e2e --headed
```

### 12.2 日志查看

```bash
# 查看最近日志
tail -100 .ai-memory/devflow.log

# 调试模式
DEVFLOW_LOG_LEVEL=debug devflow run
```

### 12.3 重置

```bash
# 清除所有记忆
devflow memory clear --force

# 重新初始化
rm -rf .ai-memory/
devflow init

# 重新研究
devflow research
```
